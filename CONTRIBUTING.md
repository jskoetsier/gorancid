# Contributing

Contributions are welcome. If you have a feature idea, bug fix, or improvement, feel free to fork the repository and submit a pull request.

## How to Contribute

1. Fork the repository
2. Create a branch for your change (`git checkout -b your-feature`)
3. Make your changes and commit them
4. Open a pull request with a clear description of what you changed and why

There are no strict requirements for commit style or branch naming — just keep things readable.

## What to Contribute

- Bug fixes
- New features or enhancements
- Documentation improvements
- Test coverage

If you are planning a larger change, opening an issue first to discuss it is appreciated but not required.

## On AI Coding Assistance

Using AI tools (such as GitHub Copilot, Claude, ChatGPT, or similar) to assist with contributions is fine. That said, please review and understand the code you submit. You are responsible for what goes into your pull request — AI-generated code can introduce bugs, security issues, or patterns that don't fit the project. Use these tools to accelerate your work, not to replace your judgment.

## Code Style

Follow the conventions already present in the codebase. Run `go fmt` and `go vet` before submitting.
